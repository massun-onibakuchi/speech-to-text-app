Autonomously garden docs and remove technical debt.

Goal:
 - Delete stale docs.
 - Refactor codebase.

Scope:
- docs/adr/
- docs/plans/
- docs/research/
- src/

Tasks:
1. Set up: Pull the latest default/base branch and switch to a fresh worktree before editing. Follow the repo workflow for worktree creation rather than editing directly on the current branch.
2. Validate: Run `pnpm run docs:validate` to inspect the doc frontmatter format.
3. Scan: Run `pnpm run docs:frontmatters` to read frontmatters.
4. Inspect: Use the frontmatters and identify discrepancies and flag documentation if the underlying code has changed.
5. Correct: Delete stale docs or change the frontmatter status field.
6. Refactor: Refactor the code to align with GOLDEN_PRINCIPLES.md
7. Reflect: Re-run validation/test after your edits. 
8. Create a PR: If the changes are worth making, create a PR and continue working until it is merged; otherwise, skip PR creation.

Constraints:
- Keep the response compact: Respect Telegram's 4096-character message limit.
- Don't change business logic.
- Don't edit docs except for the frontmatters.
- Do not stop to ask the user for approval or clarification during this autonomous flow.

Return:
- summary
- findings by file
- actions completed and short reasons
- PR status
