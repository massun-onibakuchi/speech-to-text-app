Autonomously audit and maintain controlled docs in this repository.

Scope:
- docs/decisions/
- docs/plans/
- docs/research/

Tasks:
1. Set up: Pull the latest default/base branch and switch to a fresh worktree before editing. Follow the repo workflow for worktree creation rather than editing directly on the current branch.
2. Validate: Run `pnpm run docs:validate` to inspect the doc frontmatter format.
3. Scan: Run `pnpm run docs:frontmatters` to read frontmatters.
4. Inspect: Use the frontmatters to spot stale, redundant, malformed, or misfiled docs then verify them against the codebase as needed.
5. Fix: If the findings warrant changes, keep, update, archive, delete, or rename files.
6. Reflect: Re-run validation after your edits.
7. Create a PR: If the changes are worth making, create a PR and continue working until it is merged; otherwise, skip PR creation.

Constraints:
- Keep the response compact: Respect Telegram's 4096-character message limit.
- Don't change codes. Correcting comments is allowed.
- Do not stop to ask the user for approval or clarification during this autonomous flow.
- Do not treat current frontmatter status alone as evidence that a doc still deserves to exist; re-evaluate from body content and current repo state.
- Remove temporary docs unless there is a clear reason by default.


Return:
- summary
- findings by file
- actions completed and short reasons
- PR status
